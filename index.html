Phase 1 — Month 1-3

Foundations

Master C, bare-metal programming, and basic cloud concepts

Month 1: C Language Mastery + Hardware Setup ▼

Week 1-2: C Deep Dive

  • Book: "C Programming: A Modern Approach" by K.N. King (chapters 1-20)
  • Focus areas:
    • Pointers and pointer arithmetic
    • Memory layout (stack, heap, static, text segments)
    • Bit manipulation (AND, OR, XOR, shifts, masks)
    • Structs, unions, enums
    • Function pointers and callbacks
    • Volatile, const, static keywords (critical for embedded)
    • Preprocessor macros and conditional compilation
  • Practice: 50+ problems on LeetCode (C only) — arrays, bit manipulation, linked lists
  • Tool setup: ARM GCC toolchain, OpenOCD, VS Code with C/C++ extension

Week 3-4: Hardware Acquisition + First Programs

Shopping list (total ~$50-80):

STM32F411 Nucleo board$15
ESP32-S3 DevKit$10
Raspberry Pi 4 or 5$35-60
USB Logic Analyzer$10
Breadboard + wires + LEDs$10
SPI/I2C Sensors (BME280, MPU6050)$5 ea
Project 1: Blink LED (Register-Level)
  • Read STM32F411 Reference Manual: RCC, GPIO sections
  • Understand memory-mapped I/O
  • Write to registers directly using pointers (no HAL)
  • Compile with arm-none-eabi-gcc, flash with OpenOCD
Project 2: UART Printf over Serial
  • Configure USART2 registers manually
  • Implement a basic putchar() function
  • Redirect printf to UART using _write() syscall
Month 2: Peripherals + Interrupts + Timers ▼

Week 1: GPIO + External Interrupts

Project: Button Debouncing with Interrupts
  • Configure EXTI (External Interrupt) on STM32
  • Understand NVIC (Nested Vectored Interrupt Controller)
  • Implement interrupt priorities
  • Write an ISR (Interrupt Service Routine) — keep it short
  • Learn: Interrupt latency, context saving, volatile keyword importance
  • Read: ARM Cortex-M architecture — exception model, vector table

Week 2: Timers + PWM

Project: LED Brightness Control with PWM
  • Configure TIM2/TIM3 for PWM output
  • Understand prescaler, auto-reload, capture/compare
Project: Precise Timing with SysTick
  • Implement delay_ms() using SysTick
  • Implement a basic scheduler (round-robin) using SysTick
  • Learn: Timer modes — input capture, output compare, PWM, one-pulse

Week 3: SPI Communication

  • Theory: SPI protocol — MOSI, MISO, SCLK, CS, clock polarity/phase
Project: Read BME280 over SPI
  • Read the BME280 datasheet (practice reading datasheets!)
  • Configure SPI1 registers on STM32
  • Write SPI transmit/receive functions
  • Parse multi-byte sensor data
  • Tool practice: Use logic analyzer to capture and verify SPI waveforms

Week 4: I2C Communication

  • Theory: I2C protocol — SDA, SCL, addressing, ACK/NACK, clock stretching
Project: MPU6050 Accelerometer over I2C
  • Configure I2C1 registers
  • Implement I2C start, stop, read, write functions
  • Handle I2C errors and timeouts
Project: OLED Display (SSD1306) over I2C
  • Display sensor readings on the OLED screen
  • Compare: SPI vs I2C — when to use which
Milestone: You can interface with real hardware sensors using bare-metal C.
Month 3: RTOS + DMA + Build Systems ▼

Week 1-2: FreeRTOS

  • Setup: Port FreeRTOS to STM32 (or use STM32CubeMX)
  • Concepts to learn:
    • Tasks — creation, priorities, stack size
    • Scheduler — preemptive vs cooperative
    • Queues — inter-task communication
    • Semaphores — binary and counting
    • Mutexes — priority inversion, priority inheritance
    • Software timers
    • Memory management (heap_1 through heap_5)
Project: Multi-Sensor Data Logger
  • Task 1: Read BME280 every 1 second
  • Task 2: Read MPU6050 every 100ms
  • Task 3: Print data over UART
  • Use queues to pass data between tasks
  • Use mutex to protect UART access

Week 3: DMA (Direct Memory Access)

  • Theory: DMA channels, circular mode, interrupts
Projects
  • DMA-based UART transmit (non-blocking printf)
  • DMA-based ADC sampling (continuous analog reading)
  • Why DMA matters — CPU offloading, real-time performance

Week 4: Build Systems + Version Control

  • CMake: Write CMakeLists.txt for cross-compilation, toolchain files
  • Makefile: Implicit rules, pattern rules, automatic variables
  • Git workflow: Branching (git-flow), meaningful commits, tagging releases
  • Start a GitHub portfolio — push all projects with clean READMEs

Cloud (parallel, 30 min/day)

  • Create AWS Free Tier account
  • Complete AWS Cloud Practitioner learning path (free on AWS Skill Builder)
  • Basic concepts: EC2, S3, VPC, IAM, Lambda
Milestone: 5+ GitHub projects showing bare-metal + RTOS firmware skills.
Phase 2 — Month 4-6

Intermediate

Embedded Linux, device drivers, networking protocols, cloud fundamentals

Month 4: Embedded Linux Fundamentals ▼

Week 1: Linux Internals for Embedded

  • Book: "Linux Device Drivers, 3rd Edition" (free online) — chapters 1-5
  • Kernel space vs user space
  • System calls, /dev, /sys, /proc filesystems
  • Device tree basics
  • Setup: cross-compilation toolchain (aarch64-linux-gnu-gcc)

Week 2: Buildroot

Project: Minimal Linux for Raspberry Pi
  • Configure kernel, busybox, init system
  • Add custom packages, cross-compile apps
  • Boot on Raspberry Pi, login over serial console

Week 3: Yocto Project

Project: Custom Linux Distribution
  • Set up Poky (reference distribution)
  • Write a custom recipe (.bb file)
  • Create a custom layer
  • Build and flash to Raspberry Pi
  • This is HARD — take your time, Yocto has a steep learning curve

Week 4: Device Tree

  • Theory: What device tree is, why it exists, DTS syntax
  • Project: Modify device tree to enable/disable peripherals
  • Project: Add custom device tree overlay for an SPI sensor
Month 5: Linux Device Drivers + Kernel Modules ▼

Week 1: Character Device Driver

Project: /dev/mydevice
  • Implement open, read, write, close, ioctl
  • Use copy_to_user / copy_from_user
  • Handle concurrent access with mutexes
  • Load/unload with insmod/rmmod

Week 2: Platform Driver

Project: GPIO LED Platform Driver
  • Parse device tree for GPIO pin number
  • Register as platform driver
  • Expose control via sysfs

Week 3: SPI/I2C Linux Driver

Project: Linux BME280 Driver
  • Register as SPI/I2C client driver
  • Expose sensor data via sysfs or IIO subsystem
  • Handle probe/remove lifecycle

Week 4: Debugging + Profiling

  • GDB with JTAG (OpenOCD + GDB for firmware)
  • GDB remote debugging for Linux targets
  • strace, ltrace for system call tracing
  • perf for performance profiling
  • Valgrind for memory leak detection
  • ftrace / trace-cmd for kernel tracing
Month 6: Networking Protocols + Cloud Foundations ▼

Week 1: Embedded Networking

  • TCP/IP stack basics, MQTT (pub/sub, QoS, retained messages)
  • HTTP/REST, CoAP, TLS/SSL with mbedTLS
Project: ESP32 MQTT Sensor
  • Connect to Wi-Fi
  • Publish sensor data to Mosquitto broker
  • Implement TLS for secure communication

Week 2: BLE (Bluetooth Low Energy)

  • Theory: GAP, GATT, services, characteristics, descriptors
Project: ESP32 BLE Peripheral
  • Expose sensor data as BLE characteristics
  • Test with nRF Connect mobile app
  • Advertising, connections, notifications

Week 3-4: AWS Cloud Deep Dive

  • IAM — users, roles, policies (least privilege)
  • EC2 — launch, configure, security groups
  • VPC — subnets, route tables, NAT, internet gateway
  • S3 — buckets, policies, versioning, lifecycle
  • Lambda — serverless functions in Python
  • API Gateway — RESTful APIs
  • DynamoDB — NoSQL database
  • IoT Core — MQTT broker for devices (KEY for combo)
  • CloudWatch — monitoring, alarms, logs
Project: Serverless REST API
  • Lambda + API Gateway + DynamoDB
  • Start studying for AWS Solutions Architect Associate (SAA-C03)
Milestone: You can build firmware, write Linux drivers, AND deploy cloud services.

Month 11-12: Apply + Interview Prep ▼

Target Companies

Firmware / Embedded

  • Tier 1: Apple, Google (Nest/Pixel), Meta (AR/VR), Tesla, SpaceX, NVIDIA
  • Tier 2: Qualcomm, Texas Instruments, NXP, STMicroelectronics, Analog Devices
  • Tier 3: Medtronic, Boston Scientific, John Deere, Honeywell, Bosch
  • Startups: YC's Work at a Startup — search "firmware engineer"

Cloud

  • Tier 1: AWS, Google Cloud, Microsoft Azure, Cloudflare
  • Tier 2: Any company with DevOps/SRE/Cloud team
  • Startups: Practically all need cloud engineers

IoT (Your Sweet Spot)

  • AWS IoT team, Google Nest, Particle, Samsara, Verkada
  • Industrial: Siemens, Rockwell, ABB, Schneider Electric

Apply Strategy

  • Apply to 5-10 jobs per day
  • Customize resume for each (firmware vs cloud vs IoT emphasis)
  • Reach out to recruiters directly on LinkedIn
  • Attend local/virtual meetups
  • Mock interviews: Pramp, interviewing.io (free)

Interview Preparation

Technical Questions — Firmware/Embedded

  • Volatile keyword — when and why
  • Static keyword — 3 different uses in C
  • Bit manipulation — set, clear, toggle, check bit N
  • Endianness — big vs little, conversion
  • Stack overflow — causes, detection, prevention
  • Memory-mapped I/O vs port-mapped I/O
  • Interrupt latency — factors, minimization
  • DMA vs interrupt-driven vs polling — tradeoffs
  • Priority inversion — what it is, mutex solution
  • Deadlock — conditions, prevention
  • Cache coherency (Cortex-A/Linux)
  • Power management — sleep modes, wake sources
  • Watchdog timers — purpose, implementation
  • Boot process — reset vector, startup code, main()
  • Linker script — sections, memory regions
  • Real-time — hard vs soft

Coding Questions (C)

  • Implement circular buffer
  • Implement state machine
  • Reverse bits in a byte
  • Detect overlapping memory regions
  • Implement malloc/free (fixed-pool allocator)
  • Parse protocol packet from byte stream
  • Implement software debouncing
  • Ring buffer (producer-consumer)

System Design

  • Data logger: 1kHz sampling to SD card
  • OTA update mechanism for device fleet
  • Battery-operated sensor node (power budget)

Technical Questions — Cloud

  • Scalable web app architecture on AWS
  • VPC networking — subnets, SGs, NACLs
  • CI/CD pipeline design
  • Docker vs VM — when to use which
  • Kubernetes — pods, services, deployments, ingress
  • IaC — why Terraform over console
  • Monitoring strategy — metrics, alerts
  • Cost optimization — reserved, spot, right-sizing
  • Security — IAM best practices, encryption

Final Advice

1

Build things. No amount of reading replaces writing real firmware that runs on real hardware.

2

Show your work. GitHub is your resume. Document projects with READMEs, schematics, photos.

3

Don't wait until you're "ready." Start applying at month 8-9. You'll learn from interviews.

4

The combo is rare. An engineer who can flash firmware AND deploy Terraform is unicorn-level.

5

Community matters. Join Discord servers (Embedded, Rust Embedded, AWS). Ask and answer questions.

6

Stay current. Follow RISC-V — it's the future. Learn Rust for embedded — it's growing fast.
