2. SECURITY CONSIDERATIONS (CRITICAL - READ FIRST!)
================================================================================

⚠️ CRITICAL SECURITY RULES:

1. NEVER COMMIT API KEYS TO GIT
   - Use .env files (add to .gitignore)
   - Use environment variables
   - Use encrypted secret management

2. NEVER RUN AS ROOT
   - Create dedicated user for the bot
   - Use systemd with restricted permissions
   - Apply principle of least privilege

3. FIREWALL CONFIGURATION
   - Only allow necessary outbound connections
   - Block all inbound except SSH (and only from trusted IPs)
   - Use fail2ban for SSH protection

4. API KEY SECURITY
   - Store in encrypted files or use secret management
   - Rotate keys regularly
   - Use separate keys for dev/prod

5. NETWORK SECURITY
   - Use HTTPS only
   - Verify SSL certificates
   - Use VPN or SSH tunnels for management

6. ACCESS CONTROL
   - Whitelist allowed Telegram user IDs
   - Implement rate limiting
   - Log all interactions

7. DATA PRIVACY
   - Don't log sensitive information
   - Clear conversation history regularly
   - Comply with data protection laws

8. MONITORING
   - Set up alerts for suspicious activity
   - Monitor resource usage
   - Regular security audits

sudo mkdir -p /opt/telegram-bot/{app,data,logs}
sudo chown -R telegrambot:telegrambot /opt/telegram-bot

# Deny all incoming by default
sudo ufw default deny incoming
sudo ufw default allow outgoing

# Allow SSH (change 22 to your custom port if you use one)
sudo ufw allow 22/tcp

# Enable firewall
sudo ufw enable

# Check status
sudo ufw status verbose

E. Install Fail2Ban (SSH Protection):
sudo apt install -y fail2ban
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

# Configure fail2ban
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local

# Edit jail.local to set:
# bantime = 3600
# maxretry = 3
# findtime = 600

sudo systemctl restart fail2ban

OPTION 1: OLLAMA (Recommended - Completely Free, Run Locally)
----------------------------------------------------------------
Ollama runs AI models locally on your machine. No API keys needed!

Models Available:
- Llama 3.1 (8B, 70B)
- Mistral (7B)
- Phi-3 (3.8B, 14B)
- Gemma (2B, 7B)
- And many more!

Pros:
✓ Completely free
✓ No API keys or limits
✓ Full privacy (data never leaves your machine)
✓ No internet required (after download)
✓ Fast responses

Cons:
✗ Requires more system resources (4GB+ RAM for 7B models)
✗ Initial model download required (3-10GB per model)

Installation:
curl -fsSL https://ollama.com/install.sh | sh

Start Ollama:
sudo systemctl enable ollama
sudo systemctl start ollama

Download a model:
ollama pull llama3.1:8b
# or
ollama pull mistral:7b
# or
ollama pull phi3:3.8b  # Smallest, fastest

Test it:
curl http://localhost:11434/api/generate -d '{
  "model": "llama3.1:8b",
  "prompt": "Hello! How are you?",
  "stream": false
}'

# ============================================================================
#!/usr/bin/env python3
"""
Secure Telegram AI Bot with Ollama Integration
"""

import os
import logging
from typing import Optional
from telegram import Update
from telegram.ext import (
    Application,
    CommandHandler,
    MessageHandler,
    filters,
    ContextTypes,
)
import requests
from dotenv import load_dotenv

# Load environment variables
load_dotenv('/opt/telegram-bot/.env')

# Configuration
TELEGRAM_TOKEN = os.getenv('TELEGRAM_BOT_TOKEN')
OLLAMA_HOST = os.getenv('OLLAMA_HOST', 'http://localhost:11434')
ALLOWED_USERS = [int(uid) for uid in os.getenv('ALLOWED_USER_IDS', '').split(',') if uid]
LOG_LEVEL = os.getenv('LOG_LEVEL', 'INFO')

# Configure logging
logging.basicConfig(
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    level=getattr(logging, LOG_LEVEL),
    handlers=[
        logging.FileHandler('/opt/telegram-bot/logs/bot.log'),
        logging.StreamHandler()
    ]
)
logger = logging.getLogger(__name__)

# Store conversation history per user
conversation_history = {}

def is_authorized(user_id: int) -> bool:
    """Check if user is authorized to use the bot"""
    if not ALLOWED_USERS:
        logger.warning("No allowed users configured - denying all access")
        return False
    return user_id in ALLOWED_USERS

def call_ollama(prompt: str, model: str = "llama3.1:8b") -> Optional[str]:
    """Call Ollama API for AI response"""
    try:
        response = requests.post(
            f"{OLLAMA_HOST}/api/generate",
            json={
                "model": model,
                "prompt": prompt,
                "stream": False
            },
            timeout=60
        )
        response.raise_for_status()
        return response.json().get('response', 'No response generated')
    except requests.exceptions.RequestException as e:
        logger.error(f"Ollama API error: {e}")
        return None

async def start(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """Handle /start command"""
    user_id = update.effective_user.id

    if not is_authorized(user_id):
        await update.message.reply_text(
            "⛔ Unauthorized. This bot is private."
        )
        logger.warning(f"Unauthorized access attempt by user {user_id}")
        return

    await update.message.reply_text(
        "🤖 Hello! I'm your secure AI assistant powered by Ollama.\n\n"
        "Commands:\n"
        "/start - Start conversation\n"
        "/help - Show this help\n"
        "/clear - Clear conversation history\n"
        "/model - Check current model\n\n"
        "Just send me a message and I'll respond!"
    )

async def help_command(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """Handle /help command"""
    user_id = update.effective_user.id

    if not is_authorized(user_id):
        return

    await update.message.reply_text(
        "🤖 AI Assistant Help\n\n"
        "Commands:\n"
        "/start - Start conversation\n"
        "/help - Show this help\n"
        "/clear - Clear conversation history\n"
        "/model - Check current model\n\n"
        "Features:\n"
        "✅ Powered by local Ollama (privacy-first)\n"
        "✅ Conversation history maintained\n"
        "✅ Secure access control\n"
        "✅ No data sent to external servers\n\n"
        "Just send any message and I'll respond!"
    )

async def clear_history(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """Handle /clear command"""
    user_id = update.effective_user.id

    if not is_authorized(user_id):
        return

    if user_id in conversation_history:
        del conversation_history[user_id]

    await update.message.reply_text("✅ Conversation history cleared!")

async def model_info(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """Handle /model command"""
    user_id = update.effective_user.id

    if not is_authorized(user_id):
        return

    try:
        # Get list of available models
        response = requests.get(f"{OLLAMA_HOST}/api/tags", timeout=10)
        response.raise_for_status()
        models = response.json().get('models', [])

        model_list = "\n".join([f"- {m['name']}" for m in models])

        await update.message.reply_text(
            f"🤖 Current Model: llama3.1:8b\n\n"
            f"Available Models:\n{model_list}"
        )
    except Exception as e:
        logger.error(f"Error getting model info: {e}")
        await update.message.reply_text("❌ Error getting model information")

async def handle_message(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """Handle incoming messages"""
    user_id = update.effective_user.id
    user_message = update.message.text

    # Security check
    if not is_authorized(user_id):
        await update.message.reply_text("⛔ Unauthorized access")
        logger.warning(f"Unauthorized message from user {user_id}")
        return

    # Log interaction (without sensitive content)
    logger.info(f"Message from authorized user {user_id}")

    # Show typing indicator
    await update.message.chat.send_action(action="typing")

    # Get or initialize conversation history
    if user_id not in conversation_history:
        conversation_history[user_id] = []

    # Add user message to history
    conversation_history[user_id].append(f"User: {user_message}")

    # Keep only last 10 messages for context
    if len(conversation_history[user_id]) > 10:
        conversation_history[user_id] = conversation_history[user_id][-10:]

    # Build context from history
    context_prompt = "\n".join(conversation_history[user_id])
    full_prompt = f"{context_prompt}\nAssistant:"

    # Get AI response
    ai_response = call_ollama(full_prompt)

    if ai_response:
        # Add assistant response to history
        conversation_history[user_id].append(f"Assistant: {ai_response}")
        await update.message.reply_text(ai_response)
    else:
        await update.message.reply_text(
            "❌ Sorry, I encountered an error. Please try again."
        )

def main():
    """Start the bot"""
    if not TELEGRAM_TOKEN:
        logger.error("TELEGRAM_BOT_TOKEN not set!")
        return

    if not ALLOWED_USERS:
        logger.error("ALLOWED_USER_IDS not set! Bot will deny all access.")
        return

    logger.info("Starting Telegram AI Bot...")
    logger.info(f"Allowed users: {ALLOWED_USERS}")

    # Create application
    application = Application.builder().token(TELEGRAM_TOKEN).build()

    # Register handlers
    application.add_handler(CommandHandler("start", start))
    application.add_handler(CommandHandler("help", help_command))
    application.add_handler(CommandHandler("clear", clear_history))
    application.add_handler(CommandHandler("model", model_info))
    application.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, handle_message))

    # Start bot
    logger.info("Bot is running...")
    application.run_polling(allowed_updates=Update.ALL_TYPES)

if __name__ == '__main__':
    main()
# ============================================================================

# Save and exit (Ctrl+X, Y, Enter)

# Make executable
chmod +x telegram_ai_bot.py

# Exit bot user
exit

#!/bin/bash
BACKUP_DIR="/opt/telegram-bot/backups"
DATE=$(date +%Y%m%d_%H%M%S)

E. Rate Limiting (Add to bot code):
# Add at top of telegram_ai_bot.py after imports:
from functools import wraps
from time import time

# Rate limiting decorator
user_last_request = {}
RATE_LIMIT_SECONDS = 3  # 3 seconds between requests

def rate_limit(func):
    @wraps(func)
    async def wrapper(update, context):
        user_id = update.effective_user.id
        current_time = time()

        if user_id in user_last_request:
            time_since_last = current_time - user_last_request[user_id]
            if time_since_last < RATE_LIMIT_SECONDS:
                await update.message.reply_text(
                    f"⏳ Please wait {RATE_LIMIT_SECONDS - int(time_since_last)} seconds"
                )
                return

        user_last_request[user_id] = current_time
        return await func(update, context)

    return wrapper

# Then add @rate_limit decorator to handle_message function:
@rate_limit
async def handle_message(update: Update, context: ContextTypes.DEFAULT_TYPE):
    # ... rest of function

================================================================================
9. TROUBLESHOOTING
================================================================================

PROBLEM: Bot doesn't respond
SOLUTION:
1. Check if bot service is running:
   sudo systemctl status telegram-bot.service
2. If not running, start it:
   sudo systemctl start telegram-bot.service
3. Check logs for errors:
   sudo journalctl -u telegram-bot.service -n 50
4. Verify Ollama is running:
   sudo systemctl status ollama
5. Test Ollama manually:
   curl http://localhost:11434/api/generate -d '{"model":"llama3.1:8b","prompt":"test","stream":false}'

PROBLEM: "Unauthorized" message
SOLUTION:
1. Get your Telegram user ID from @userinfobot
2. Add to .env file:
   sudo su - telegrambot
   nano /opt/telegram-bot/.env
   # Update ALLOWED_USER_IDS with your ID
3. Restart bot:
   exit
   sudo systemctl restart telegram-bot.service

PROBLEM: Slow responses
SOLUTION:
1. Check if model is appropriate for your RAM:
   ollama list
2. Switch to smaller model:
   ollama pull phi3:3.8b
3. Update bot code to use smaller model
4. Check system resources:
   htop

PROBLEM: Model not found error
SOLUTION:
1. List installed models:
   ollama list
2. Pull the model:
   ollama pull llama3.1:8b
3. Restart bot:
   sudo systemctl restart telegram-bot.service

PROBLEM: Out of memory
SOLUTION:
1. Switch to smaller model (phi3:3.8b instead of llama3.1:8b)
2. Increase swap space:
   sudo fallocate -l 4G /swapfile
   sudo chmod 600 /swapfile
   sudo mkswap /swapfile
   sudo swapon /swapfile
3. Add to /etc/fstab for persistence:
   /swapfile none swap sw 0 0

PROBLEM: Network errors
SOLUTION:
1. Check internet connection:
   ping -c 3 telegram.org
2. Check firewall:
   sudo ufw status
3. Verify outbound HTTPS is allowed:
   curl -I https://api.telegram.org
4. Check DNS:
   nslookup telegram.org

PROBLEM: Bot stops after system reboot
SOLUTION:
1. Enable auto-start:
   sudo systemctl enable telegram-bot.service
2. Check dependencies:
   sudo systemctl enable ollama.service
3. Verify service started:
   sudo systemctl status telegram-bot.service

PROBLEM: Permission denied errors
SOLUTION:
1. Fix ownership:
   sudo chown -R telegrambot:telegrambot /opt/telegram-bot
2. Fix permissions:
   sudo chmod 700 /opt/telegram-bot
   sudo chmod 600 /opt/telegram-bot/.env
3. Restart service:
   sudo systemctl restart telegram-bot.service

PROBLEM: API token invalid
SOLUTION:
1. Get new token from @BotFather:
   /newbot or /token
2. Update .env file:
   sudo su - telegrambot
   nano /opt/telegram-bot/.env
3. Restart bot:
   exit
   sudo systemctl restart telegram-bot.service

PROBLEM: Logs filling up disk
SOLUTION:
1. Configure log rotation (see section 7D)
2. Clear old logs:
   sudo journalctl --vacuum-time=7d
3. Compress old log files:
   cd /opt/telegram-bot/logs
   gzip *.log.1 *.log.2

================================================================================
10. BACKUP & RECOVERY
================================================================================

A. What to Backup:
1. Configuration files (.env)
2. Bot code (telegram_ai_bot.py)
3. Logs (optional, for audit trail)
4. Systemd service file

B. Manual Backup:
# Create backup directory
mkdir -p ~/telegram-bot-backup

# Backup configuration (encrypted)
sudo cp /opt/telegram-bot/.env ~/telegram-bot-backup/
gpg --symmetric --cipher-algo AES256 ~/telegram-bot-backup/.env
rm ~/telegram-bot-backup/.env

# Backup bot code
sudo cp /opt/telegram-bot/app/telegram_ai_bot.py ~/telegram-bot-backup/

# Backup service file
sudo cp /etc/systemd/system/telegram-bot.service ~/telegram-bot-backup/

# Create tarball
tar -czf telegram-bot-backup-$(date +%Y%m%d).tar.gz ~/telegram-bot-backup/

# Store securely (off-system)

C. Automated Backup (see section 7C):
Backups run daily at 2 AM, encrypted with GPG

D. Restore from Backup:
# Extract backup
tar -xzf telegram-bot-backup-YYYYMMDD.tar.gz

# Decrypt configuration
gpg -d config_YYYYMMDD_HHMMSS.tar.gz.gpg > config.tar.gz
tar -xzf config.tar.gz

# Restore files
sudo cp telegram-bot-backup/.env /opt/telegram-bot/
sudo cp telegram-bot-backup/telegram_ai_bot.py /opt/telegram-bot/app/
sudo cp telegram-bot-backup/telegram-bot.service /etc/systemd/system/

# Fix permissions
sudo chown -R telegrambot:telegrambot /opt/telegram-bot
sudo chmod 600 /opt/telegram-bot/.env

# Reload and restart
sudo systemctl daemon-reload
sudo systemctl restart telegram-bot.service

E. Disaster Recovery Plan:
1. Keep backups on separate device/cloud
2. Document recovery procedure
3. Test restore process regularly
4. Keep list of installed packages:
   dpkg --get-selections > package-list.txt
5. Store Ollama model names (models can be re-downloaded)

F. Migration to New Server:
1. Install Ollama on new server
2. Download models: ollama pull llama3.1:8b
3. Restore backup files
4. Install Python dependencies
5. Configure systemd service
6. Test thoroughly before switching

def call_groq(prompt: str) -> Optional[str]:
    """Call Groq API"""
    try:
        from groq import Groq
        client = Groq(api_key=os.getenv('GROQ_API_KEY'))

        response = client.chat.completions.create(
            model="llama-3.1-8b-instant",
            messages=[{"role": "user", "content": prompt}],
            temperature=0.7,
            max_tokens=1024,
        )
        return response.choices[0].message.content
    except Exception as e:
        logger.error(f"Groq API error: {e}")
        return None
